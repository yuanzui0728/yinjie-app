# iOS Device Regression Checklist

日期：2026-04-06
目标：在真机上逐项验证 iOS 提审前的关键路径

## A. 启动与环境

1. 首次安装后启动 App
2. Safe Area 是否正确
3. Splash 是否正常消失
4. Runtime config 是否正确注入远程 Core API 地址
5. 服务不可达时提示是否清晰

## B. 账号与会话

1. Onboarding
2. Login
3. 退出当前世界
4. 退出全部设备
5. 删除账号
6. 杀进程后 session 是否恢复
7. 原生安全存储 plugin 接通后 token 是否仍能恢复

## C. 聊天

1. 会话列表加载
2. 单聊发送消息
3. 单聊接收回复
4. 举报当前对话对象
5. 群聊发送消息
6. 群聊异步回复
7. 输入框在键盘弹起时不被遮挡
8. 新消息是否自动滚到底部

## D. 社交与内容

1. 朋友圈发帖
2. 朋友圈点赞/评论
3. 发现页发帖
4. 发现页摇一摇
5. 场景触发好友申请
6. 举报角色
7. 屏蔽角色
8. 屏蔽后目标是否从发现/通讯录/会话入口消失
9. 解除屏蔽

## E. 法务与审核

1. 隐私政策页可打开
2. 用户协议页可打开
3. 社区与安全说明页可打开
4. 资料页可看到最近举报记录
5. 资料页可看到屏蔽名单

## F. 设备状态切换

1. 切前后台
2. 锁屏再恢复
3. 弱网切换
4. 断网再恢复
5. 推送点击回到 App

## 记录字段

每个用例建议记录：

1. Device Model
2. iOS Version
3. Build Version
4. Result
5. Notes
