# Phase 0 - UI / UX Baseline

## 主产品必须保留的体验特征

- 窄屏社交壳，而不是传统后台布局。
- `Splash -> Onboarding -> Tabs` 的情绪入场路径保留。
- 聊天页的即时感、typing 感、系统提示感保留。
- 朋友圈与发现页继续沿用熟悉的社交心智，不改成信息流后台。
- Profile、角色详情页继续保持“角色存在感”而不是表单感。

## 允许重构但不能改变目的的部分

- 页面结构可以重写为 feature-based。
- 样式系统可以切到 shared design system。
- 动效可以统一，但不能丢掉叙事入场和社交沉浸感。
- 桌面适配可以增强，但不能把主产品做成纯桌面工作台。

## 新增 UX 目标

- 首次启动改为桌面化配置向导。
- 主产品新增系统状态感知，但不打断社交体验。
- 后台成为正式本地控制台：角色管理、模型配置、日志、诊断、备份恢复。
