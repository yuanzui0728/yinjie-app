# 桌面端重构方案：从“手机壳”改为真正的桌面工作台

## Summary
目标是把 Windows / macOS 桌面端从“复用手机单列页面”重构为“桌面专属信息架构”，风格延续现有隐界品牌，但交互和布局按桌面效率重做，主形态参考微信桌面端。

本次范围覆盖全链路：
- 启动 / Setup / 登录 / Onboarding 做桌面化入口
- 登录后主界面改为微信式三栏工作台
- 聊天作为桌面主场景
- 通讯录、朋友圈、发现、我 保留，但都使用桌面专属布局
- 首版保持单窗口，不做会话新开窗
- 朋友圈 / 发现保留为独立桌面页，不并入聊天工作台
- 第三栏首版固定为“会话资料栏”，不做多会话并行

## Key Changes
### 1. 平台壳层拆分
在 `apps/app/src/features/shell` 建立平台分流，不再让 `AppShell` 无条件返回 `MobileShell`。
- 新增桌面壳层 `DesktopShell`
- 保留现有 `MobileShell` 仅服务移动端 / 窄屏
- `RootLayout` 按 `runtimeContext.channel` 或桌面运行时能力切换桌面 / 移动壳层
- 桌面壳层负责全局背景、窗口级留白、三栏容器、桌面导航、顶部窗口操作区预留
- 移动端现有底部 Tab 和单列页面行为保持不变

### 2. 路由与页面编排调整
保持现有业务路由语义，新增桌面专属页面编排层，而不是重写数据层。
- `/tabs/chat` 在桌面端不再只是“会话列表页”，而是桌面主工作台入口
- 桌面聊天工作台固定三栏：
  - 左栏：主导航，含消息 / 通讯录 / 朋友圈 / 发现 / 我
  - 中栏：当前模块的列表区；在消息场景显示会话列表
  - 右栏：主内容区；在消息场景显示聊天内容，并附带资料栏
- 详情路由如 `/chat/$conversationId`、`/character/$characterId`、`/group/$groupId` 在桌面端优先表现为工作台右侧内容切换，而不是整页跳转感
- 移动端继续保留当前整页跳转路径，避免双端互相污染
- Setup / Login / Onboarding 在桌面端改为更宽的居中双区或单区卡片布局，不再限制在手机壳宽度内

### 3. 桌面聊天工作台
消息模块按“微信桌面端”方式重构为桌面核心场景。
- 会话列表支持更高信息密度：头像、标题、最后一条消息、时间、未读、置顶位预留
- 聊天区常驻右侧，不再进入一个独立手机页
- 当前聊天头部展示会话标题、类型、成员数、快捷入口
- 第三栏为会话资料栏，首版包含：
  - 会话基础信息
  - 成员列表 / 角色摘要
  - 快捷动作入口占位（如查看联系人、群信息、上下文）
- 聊天输入区、消息流、Socket 逻辑、已读逻辑继续复用现有数据和动作
- 聊天列表与聊天详情拆成可组合的桌面视图组件，避免把当前 `chat-list-page` / `chat-room-page` 直接硬拼

### 4. 其他模块的桌面化
桌面端其余模块不再沿用手机单列卡片堆叠。
- 通讯录：
  - 左侧为联系人分组 / 入口
  - 中间为联系人列表
  - 右侧为角色详情或操作面板
- 朋友圈：
  - 主列展示动态流
  - 侧栏展示发布入口、个人摘要或筛选信息
- 发现：
  - 把“随机相遇 / 场景相遇 / 广场动态”拆成桌面分区
  - 顶部操作区更宽，动态流与互动区分栏展示
- 我：
  - 账号资料、API Key、运行时信息拆成多个桌面面板
  - `DesktopRuntimePanel` 继续保留，但从“手机长页中的一个 section”改成桌面诊断卡片区

### 5. 组件与接口层
主要新增的是界面编排接口，不改后端 API。
- 新增桌面壳层与工作台组件族：
  - `DesktopShell`
  - `DesktopWorkspaceLayout`
  - `DesktopNavRail`
  - `DesktopListPane`
  - `DesktopContentPane`
  - `DesktopInspectorPane`
- 现有页面拆为“数据容器 + 视图片段”模式：
  - 把会话列表、聊天消息区、资料栏拆成独立可组合组件
  - 让移动端页面继续消费这些能力，但保持移动端外观
- 必要时新增前端类型：
  - `DesktopWorkspaceSection`
  - `DesktopWorkspaceContext`
  - `DesktopInspectorTab`
- 不新增后端路由、不改 contracts wire shape、不做数据库变更
- 若实现中出现导航状态需求，优先放在前端路由 search params 或本地 store，不引入新服务接口

## Test Plan
不新增测试文件，验证以现有检查和人工走查为主。
- `apps/app` 能通过类型检查与构建
- 桌面运行时下，启动后进入桌面壳层而不是 `max-w-[460px]` 手机容器
- Setup / Login / Onboarding 在桌面端使用宽布局，移动端维持现状
- 消息模块：
  - 默认进入三栏工作台
  - 选中会话后右侧即时展示聊天
  - Socket 新消息、输入发送、未读更新、已读标记不回归
- 通讯录：
  - 发起会话后能在桌面工作台内切到聊天内容
- 朋友圈 / 发现 / 我：
  - 各自进入桌面独立页
  - 页面在 1280px+ 和 1440px+ 下没有明显空洞或手机拉伸感
- 移动端 / Web 窄屏：
  - 继续走现有 `MobileShell`
  - 底部 Tab、整页跳转、聊天页行为不回归
- Windows / macOS：
  - 同一套桌面前端布局可用
  - 不依赖平台专属 UI 分叉
- 回归验证：
  - 桌面运行时守卫、桌面本地 Core API 控制、Profile 中运行时面板继续可用

## Assumptions
- 设计基调延续当前隐界品牌，只重构桌面布局与交互密度，不另起一套视觉体系
- 首版不做多窗口，不做会话标签并行，不做原生窗口管理增强
- 首版右侧第三栏固定为会话资料栏，不承担并行会话能力
- 保持当前路由主干，优先做平台条件渲染和桌面工作台编排，避免一次性重写整个导航系统
