# Web 端电脑版好友详情页与聊天页对齐微信电脑端规划

## 任务目标
- 盘清当前 `apps/app` 里桌面态好友详情页、桌面聊天页，与微信电脑端在功能、操作、信息架构上的差异。
- 明确哪些地方已经接近微信，哪些地方还只是“像个聊天 App”，避免重复误判现状。
- 给出一条“尽量按微信电脑端原操作习惯复刻”的实施顺序。

### 微信电脑端近期开口径
- 2026-03-19，IT 之家转引“微信派”称微信 `4.1.8` 电脑端支持语音输入、一键撤回、视频倍速。
- 2026-03-19，PChome 同步提到 `Ctrl + Win` 语音输入、批量撤回图片 / 视频 / 文件等能力。
- 这些信息说明：
  - 微信电脑端现在不是“纯收消息工具”，而是高频办公主端。
  - 桌面端聊天页必须把消息处理、右键、多选、文件、提醒、输入效率当主线，而不是附属功能。

## 差异总判断

### 结论
- 现在的桌面聊天页已经有一部分微信式“操作骨架”，但页面质感、信息组织、动作顺序、联系人资料联动仍然不对。
- 最大问题不是“功能按钮有没有”，而是：
  - 好友详情页还是偏“角色资料卡”
  - 聊天页还是偏“项目自定义聊天工作区”
  - 联系人详情和聊天信息面板没有形成微信那种一致的认知路径

## 一、好友详情页和微信电脑端的差异

### 1. 信息架构不对
- 当前桌面通讯录右侧详情 `contact-detail-pane` 仍然是“大卡片 + 多分区说明页”。
- 微信电脑端联系人详情更像“资料页”：
  - 顶部联系人身份
  - 中段联系人字段
  - 下段内容入口
  - 底部管理开关 / 危险操作
- 现在的页面仍混入太多“角色世界”字段：
  - `relationship`
  - `bio`
  - `最近互动`
  - 这些放在首屏会削弱微信联系人感。

### 2. 首屏主操作顺序不对
- 当前首屏右上主按钮是 `发消息` 或 `查看资料`，但微信桌面联系人详情更强调：
  - 发消息
  - 音视频入口
  - 资料行点击进入二级操作
- 现在桌面联系人详情没有形成“联系人管理入口优先、角色档案次级”的层级。

### 3. 字段排布不够微信
- 当前字段有：
  - 备注
  - 昵称
  - 地区
  - 来源
  - 标签
  - 隐界号
  - 个性签名
  - 最近互动
- 问题不是字段缺得多，而是排法不像微信：
  - 微信更强调备注、昵称、微信号式唯一标识、地区、来源、标签
  - 当前把 `最近互动`、长段 `bio`、角色身份说明塞进主页面，视觉重心偏了

### 4. 编辑方式不对
- 当前在详情页里直接展开可编辑表单。
- 微信电脑端联系人详情更常见的是：
  - 点击某一项进入轻量编辑
  - 或单独弹层编辑备注 / 标签
- 现在整块展开输入框，更像后台资料页，不像微信联系人页。

### 5. 内容入口还不够微信式
- 当前只有：
  - `更多资料`
  - `朋友圈` 占位
  - `共同群聊` 文本
- 微信桌面联系人详情更强调可点击入口行：
  - 朋友圈
  - 共同群聊
  - 更多信息
  - 投诉 / 拉黑 / 删除联系人等管理操作
- 现在“共同群聊”只是附加信息，不是明确入口。

### 6. 完整好友详情路由不该继续走移动端思路
- `character-detail-page.tsx` 现在更像手机版“朋友信息”页搬到桌面宽度下显示。
- 这页保留没问题，但它不应该成为桌面主路径。
- 微信电脑端主路径应该是：
  - 通讯录右侧资料页
  - 聊天右侧聊天信息面板
  - 而不是打开一个全屏朋友资料页再操作

## 二、聊天页和微信电脑端的差异

### 1. 视觉和空间气质不对
- 当前桌面聊天页大量使用暖色渐变、柔和阴影、圆角卡片。
- 微信电脑端更克制：
  - 冷白灰底
  - 更平的层次
  - 更少装饰感
- 这一点会直接导致“功能差不多，但第一眼不像微信”。

### 2. 聊天头部还不够微信标准件
- 当前头部已经有：
  - `聊天记录`
  - `通话`
  - `更多`
- 方向是对的，但差异仍在：
  - 头部高度、留白、按钮触感还是项目自定义
  - 通话现在是跳到 `/desktop/mobile` 的接力页，不是真桌面通话
  - 微信电脑端这里更像原生桌面工具栏，而不是 Web 控件组

### 3. 单聊右侧“聊天信息”面板仍然更像设置面板，不像微信联系人信息
- 当前单聊侧栏结构：
  - 顶部头像和关系
  - 聊天成员
  - 常用
  - 设置
  - 更多
- 但微信电脑端单聊信息面板更接近“联系人卡 + 聊天管理”：
  - 备注 / 昵称识别更强
  - 联系人资料入口更明确
  - 媒体 / 文件 / 搜索更靠前
  - 删除、拉黑等危险动作更收束
- 现在侧栏缺少联系人资料字段联动，和通讯录详情像两套系统。

### 4. 聊天信息面板缺少和好友详情页一致的数据语义
- 通讯录详情页已经有：
  - 备注
  - 地区
  - 来源
  - 标签
  - 共同群聊
- 聊天信息面板只有：
  - 关系文案
  - 添加到通讯录
  - 背景设置
  - 清空 / 删除 / 投诉 / 拉黑
- 微信电脑端不会让用户在“通讯录里看到的是一套人设，在聊天里看到的是另一套人设”。
- 当前最大缺口是：
  - 聊天侧栏没有复用联系人资料主字段
  - 通讯录页和聊天侧栏缺少共享组件与共享排序

### 5. 媒体与资料聚合还不够微信
- 当前有：
  - `查找聊天记录`
  - `聊天文件`
- 但微信电脑端在聊天信息面板里，用户期待的是更强的“聊天资料聚合”：
  - 图片 / 视频 / 文件 / 链接等更明确的分类入口
  - 更少跳页面，更多在当前聊天上下文完成处理
- 现状仍偏“跳到另一个桌面工作区页”。

### 6. 右键菜单能力有了，但菜单语言和顺序还不够微信
- 当前消息右键已经很全。
- 主要差异变成：
  - 菜单文案不完全是微信习惯
  - 菜单顺序未按微信高频优先级排列
  - 不同消息类型的差异化动作还不够微信化
- 所以消息区不再是“从 0 到 1”，而是“把 70 分修到 95 分”。

### 7. 桌面专属效率能力还不够
- 微信电脑端的强感知能力包括：
  - 真正顺手的语音输入
  - 一键撤回批量消息
  - 更成熟的文件处理
  - 聊天记录备份 / 管理
- 当前项目虽然有语音转写入口和聊天记录页，但桌面“办公主端”感还不够强。

## 三、最核心的结构性差异

### 当前是三套页面逻辑
- 通讯录右侧好友详情页一套逻辑
- 全屏好友详情路由一套逻辑
- 聊天右侧聊天信息一套逻辑

### 微信电脑端更像两套主界面
- 主界面 1：通讯录联系人资料
- 主界面 2：聊天页 + 聊天信息侧栏
- 二者字段、操作、视觉规则高度统一

### 必须收口的方向
- 桌面不要再把 `character-detail-page.tsx` 当主承载。
- 桌面应收口为：
  - `ContactDetailPane` 负责完整联系人资料
  - `DesktopChatDetailsPanel` 负责聊天内侧栏版联系人管理
- 两者共享：
  - 顶部身份模块
  - 联系人字段顺序
  - 操作文案
  - 危险操作顺序

## 目标方案：按微信电脑端“尽量照抄”

## Phase 1：先统一信息架构和视觉骨架
- 目标：用户一眼看上去就像微信电脑端，而不是先看到项目风格。
- 要做的事：
  - 桌面聊天区去暖色渐变，改为微信式灰白基底
  - 通讯录详情页去大卡片化，改为联系人资料页结构
  - 聊天信息面板弱化“设置页”感，强化“联系人信息 + 聊天管理”

### Phase 1 页面结构要求
- 通讯录好友详情：
  - 顶部头像、备注名、昵称、隐界号
  - 中部字段行
  - 下部内容入口行
  - 底部管理开关和危险动作
- 聊天页：
  - 头部只保留微信式标题 + 工具按钮
  - 主消息区减少装饰
  - 右侧聊天信息面板改成更接近微信的行式信息块

## Phase 2：统一好友详情与聊天信息面板的数据语义
- 目标：同一个联系人，在通讯录和聊天里看起来是同一个人。
- 要做的事：
  - 抽一套桌面联系人基础资料组件
  - 两处统一字段顺序：
    - 备注
    - 昵称
    - 隐界号
    - 地区
    - 来源
    - 标签
    - 个性签名
  - 聊天信息面板新增：
    - 查看联系人资料
    - 共同群聊
    - 标签摘要
  - 通讯录详情保留完整视图，聊天侧栏保留压缩视图

### Phase 2 推荐拆分
- 新增 `apps/app/src/features/contacts/desktop-contact-profile-summary.tsx`
- 新增 `apps/app/src/features/contacts/desktop-contact-info-rows.tsx`
- 新增 `apps/app/src/features/contacts/desktop-contact-management-rows.tsx`
- `ContactDetailPane` 和 `DesktopChatDetailsPanel` 共用这些组件

## Phase 3：把操作路径抄到微信顺序
- 目标：让用户用肌肉记忆而不是重新学习。
- 要做的事：
  - 通讯录详情页主按钮顺序调整为：
    - `发消息`
    - 语音 / 视频入口占位或灰态
    - `更多资料` 降级为行入口，不再做大主按钮
  - 聊天信息面板顺序调整为：
    - 查找聊天记录
    - 聊天文件 / 聊天资料
    - 消息免打扰
    - 置顶聊天
    - 查看联系人资料
    - 黑名单 / 删除聊天
  - 危险操作统一放底部，避免散落在不同 section

## Phase 4：把已有能力修成微信语言，而不是再造新功能
- 目标：优先把现有 70 分能力做成 95 分微信感。
- 要做的事：
  - 重排消息右键菜单顺序
  - 对图片 / 文件 / 名片 / 位置卡片给出更微信式动作命名
  - 调整会话右键菜单层级和危险项位置
  - 优化聊天头部按钮 hover / active / spacing

### 这一阶段不要重复做的事
- 不要再新造一套消息菜单体系。
- 不要把桌面好友详情继续做成“角色世界资料展示页”。
- 不要把“查看资料”当桌面主闭环。

## Phase 5：桌面专属能力补齐
- 目标：补足微信电脑端“办公主端”感。
- 建议项：
  - 真桌面通话能力，而不是跳手机接力页
  - 聊天资料聚合入口：图片 / 视频 / 文件 / 链接
  - 更强的聊天记录管理和回跳
  - 语音输入入口与快捷键提示更显性

## 建议实施顺序

### P0，本周先做
- 改 `ContactDetailPane` 结构和样式
- 改 `DesktopChatDetailsPanel` 单聊信息结构
- 抽共享联系人资料组件
- 调整 `ConversationThreadPanel` 的桌面视觉基底

### P1，接着做
- 调整消息右键菜单顺序、文案、类型化动作
- 调整聊天头部按钮和右侧侧栏文案
- 让“查看联系人资料”在聊天侧栏里成为明确入口

### P2，之后做
- 补聊天资料聚合
- 补更强桌面通话 / 语音输入 / 记录管理闭环

## 明确不建议的路线
- 不建议继续在桌面主流程里强化 `character-detail-page.tsx`。
- 不建议把 `bio`、`expertDomains`、`relationship` 这类角色信息继续放在好友详情首屏主位。
- 不建议再按旧认知把消息右键、多选、拖拽发送当成“还没做”；这些已经做了，应该进入微信化精修阶段。

## 一句话方案
- 通讯录页抄微信“联系人资料页”。
- 聊天页抄微信“消息主界面 + 聊天信息侧栏”。
- 两边共用一套联系人字段和操作顺序。
- 先修信息架构和页面气质，再修细节菜单，最后补桌面专属高级能力。
