# Web 端桌面版聊天页三个点与聊天信息完全对齐微信规划

## 任务目标

- Web 端桌面版聊天页右上角“聊天信息”入口从当前 `Info` 图标改为微信电脑端风格的 `...`。
- 点击 `...` 后的承载方式、排版顺序、开合逻辑、单聊/群聊差异，整体按微信电脑端处理收口。
- 范围只覆盖 `apps/app` 的桌面聊天工作区，不改移动端，不改后端接口，不新增测试文件。

## 参考基线

### 外部产品基线

- 2026-03-13 的公开体验文章显示，微信 `4.1.8 for Windows / Mac` 已经把语音/视频通话入口统一收进聊天页右上角，并且整套聊天头部与表面视觉更接近纯白、扁平、弱装饰风格。
- 对“右上角三个点后具体承载什么”这件事，公开文章能确认的是整体改版方向，逐项字段并不完整；本规划按当前微信电脑端稳定认知和用户要求执行：
  - `...` 是聊天信息入口，不是跳新页面。
  - 点击后应在当前聊天上下文里展开右侧信息面板。
  - 交互以“微信桌面端肌肉记忆一致”为优先级，不保留项目自己的另一套入口逻辑。

## 当前现状

### 1. 入口图标不对

- 当前桌面聊天头部动作区是：
  - `查找聊天记录`
  - `通话`
  - `聊天信息`
- 其中“聊天信息”使用的是 `Info` 图标，不是微信电脑端的 `...`。

### 2. 入口逻辑不对

- 单聊头部标题整个可点击，点标题本身也会直接打开聊天信息侧栏。
- 群聊头部标题和成员数区域也承担了打开聊天信息/成员搜索的职责。
- 这会造成“标题区”和“右上角信息按钮”双入口并存，不符合微信电脑端把聊天信息入口收口到 `...` 的使用习惯。

### 3. 侧栏容器样式不对

- 当前 `desktop-chat-side-panel.tsx` 是通用项目侧栏：
  - 顶部有“聊天记录面板 / 聊天信息面板”文案
  - 有独立关闭按钮
  - 面板外观偏项目自定义玻璃态和浮层感
- 微信电脑端右侧信息区更像聊天窗口的一部分，而不是带“工具面板标题”的产品内检查器。

### 4. 单聊信息面板内容结构不对

- 当前单聊面板更像“联系人资料页压缩版”，信息块包括：
  - 联系人资料
  - 聊天内容
  - 内容与关系
  - 聊天设置
  - 更多
- 里面夹带了不少当前项目语义，例如：
  - `隐界号`
  - `关系摘要`
  - `更多资料`
  - `投诉`
- 这些不一定都该消失，但不能继续以当前这套卡片式资料页结构承载，否则第一眼就不像微信。

### 5. 群聊信息面板有能力，但仍不是微信处理

- 当前群聊侧栏已经有：
  - 成员宫格
  - 搜索成员
  - 群名/群公告/群二维码
  - 查找聊天记录/聊天文件
  - 免打扰及群设置
- 但具体排布仍偏项目自定义：
  - 模块标题和卡片边界太重
  - “群工具”等分组偏功能堆叠
  - 还有若干非微信优先路径

## 目标口径

### 总原则

- 这次不是“把图标换成三个点就结束”，而是把桌面聊天页里“聊天信息”这条完整链路改成微信电脑端语义。
- 优先复制：
  - 入口位置
  - 入口数量
  - 开合逻辑
  - 右侧信息面板的分区顺序
  - 单聊 / 群聊的差异化信息密度
- 不复制：
  - 微信品牌资源
  - 官方素材
  - 任何受版权保护的图像资产

### 目标行为

- 桌面聊天页右上角仍保留三项动作区：
  - `搜索聊天记录`
  - `通话`
  - `更多`
- “更多”按钮使用横向三个点图形，而不是 `Info`。
- `...` 是桌面聊天信息的唯一主入口。
- 标题区不再承担“打开聊天信息”职责，避免双入口。

## 交互规划

### 1. 头部动作区

- 单聊和群聊统一复用同一套桌面头部动作组件。
- `更多` 按钮图标替换为微信电脑端风格的 `...`。
- 按钮视觉改为更克制的微信式 hover/active：
  - 默认弱对比
  - hover 只有轻微底色
  - active 不出现当前这类较重的白卡片和阴影

### 2. 标题区点击行为收口

- 单聊：
  - 标题只负责展示会话名和副标题，不再点击打开聊天信息。
- 群聊：
  - 群名和人数只做展示，不再直接承担“打开群聊信息/成员搜索”。
- 所有进入聊天信息的桌面端入口统一收口到 `...`。

### 3. `...` 的开合逻辑

- 点击 `...`：
  - 若当前未打开右栏，则打开聊天信息栏。
  - 若当前已打开聊天信息栏，则关闭。
  - 若当前打开的是聊天记录栏，则切换到聊天信息栏。
- 点击“搜索聊天记录”按钮：
  - 若当前未打开右栏，则打开聊天记录栏。
  - 若当前已打开聊天记录栏，则关闭。
  - 若当前打开的是聊天信息栏，则切换到聊天记录栏。
- 切换会话时：
  - 右栏保持当前模式不变，内容跟随切到新会话刷新。
  - 这是更接近微信桌面端工作流的处理，也能减少反复开关面板。

### 4. 关闭逻辑

- 保留右栏顶部关闭能力，但不要继续用“工具面板”语义。
- 关闭按钮文案和层级都要弱化，只作为右栏自身的一部分存在。
- 不新增遮罩，不做 Drawer 式覆盖动画，而是继续采用微信式“右侧占位展开/收起”。

## 排版规划

### A. 右栏容器

- 取消当前 `聊天记录面板 / 聊天信息面板` 的显性产品标题。
- 右栏头部只保留当前会话名称和简要上下文，不再出现“面板”字样。
- 整个右栏改成更平、更薄边界的微信式信息列：
  - 顶部固定
  - 内容区滚动
  - 分区之间以留白和细分隔线区分，而不是大卡片套小卡片

### B. 单聊信息面板

- 顶部资料区：
  - 头像
  - 昵称
  - 备注或关系文案
  - 必要时保留极少量实例特有字段，但不放在首屏主区
- 第一屏高频区：
  - 查找聊天记录
  - 聊天文件
  - 聊天背景
- 设置区：
  - 消息免打扰
  - 置顶聊天
  - 添加到通讯录（仅未加好友场景）
- 风险区：
  - 加入黑名单
  - 删除聊天
  - 清空聊天记录
- 低频扩展项如 `投诉`、`隐界号`、`共同群聊` 不应继续抢占首屏主区，建议下沉到次级信息区或折叠处理。

### C. 群聊信息面板

- 顶部群资料区：
  - 群头像
  - 群名
  - 成员数
- 成员宫格区：
  - 成员头像栅格
  - `+` / `-` 入口
  - 查看全部成员
- 高频区：
  - 群公告
  - 查找聊天记录
  - 聊天文件
  - 群二维码
- 设置区：
  - 消息免打扰
  - 置顶聊天
  - 我在本群的昵称
  - 显示群成员昵称
  - 保存到通讯录
  - 聊天背景
- 风险区：
  - 清空聊天记录
  - 删除并退出
- 现有“群工具”等偏项目堆叠分组不作为微信主线布局的一部分，能下沉就下沉。

## 实施方案

### Phase 1：头部入口微信化

目标：先把用户第一眼最敏感的右上角入口改对。

涉及文件：

- `apps/app/src/features/desktop/chat/desktop-chat-header-actions.tsx`
- `apps/app/src/features/chat/conversation-thread-panel.tsx`
- `apps/app/src/features/chat/group-chat-thread-panel.tsx`

实施点：

- `DesktopChatHeaderActions` 中把 `Info` 替换为 `Ellipsis` 或更接近微信的三点图形。
- 保留搜索和通话入口，不额外发明第四个入口。
- 单聊/群聊标题区从 `button` 改成纯展示容器，移除“点击标题打开聊天信息”的行为。

### Phase 2：右栏容器去“工具面板化”

目标：让侧栏外观先像微信，再承接里面的信息结构。

涉及文件：

- `apps/app/src/features/desktop/chat/desktop-chat-side-panel.tsx`
- `apps/app/src/features/desktop/chat/desktop-chat-workspace.tsx`

实施点：

- 保留现有 `rightPanelMode: "history" | "details" | null` 状态模型，不需要重做状态层。
- 只重构容器样式和头部文案：
  - 去掉“聊天信息面板/聊天记录面板”字样
  - 收弱关闭按钮
  - 调整边框、背景、阴影、分隔
- 切会话时继续复用现有模式跟随逻辑。

### Phase 3：单聊信息面板重排

目标：把当前“联系人资料页压缩版”改成微信单聊信息面板。

涉及文件：

- `apps/app/src/features/desktop/chat/desktop-chat-details-panel.tsx`
- 视情况新增内部子组件，例如：
  - `desktop-chat-details-section.tsx`
  - `desktop-chat-details-row.tsx`

实施点：

- 复用当前已有 mutation 和查询，不改接口：
  - `setConversationPinned`
  - `setConversationMuted`
  - `clearConversationHistory`
  - `hideConversation`
  - `blockCharacter`
- 主要改的是信息架构、排序和视觉层级，不是重写业务链路。
- 低频实例字段改为弱展示，避免破坏微信式首屏结构。

### Phase 4：群聊信息面板重排

目标：把当前群聊面板从“功能集合页”收成微信群聊信息面板。

涉及文件：

- `apps/app/src/features/desktop/chat/desktop-chat-details-panel.tsx`

实施点：

- 继续复用现有能力：
  - `getGroup`
  - `getGroupMembers`
  - `updateGroup`
  - `updateGroupPreferences`
  - `updateGroupOwnerProfile`
  - `clearGroupMessages`
  - `leaveGroup`
- 重点不是补新接口，而是调整：
  - 首屏信息排序
  - 群成员区视觉
  - 设置区和危险区层级
  - 非微信主线功能的下沉位置

## 不做项

- 这轮不改移动端详情页。
- 这轮不新增新的桌面聊天路由。
- 这轮不改后端接口和数据库结构。
- 这轮不做“完全像微信”的品牌资源复刻，只做交互和信息架构对齐。

## 验收口径

- 桌面单聊和群聊右上角“聊天信息”入口都显示为 `...`，不再出现 `Info` 图标。
- 点击标题区不再打开聊天信息。
- 点击 `...` 会在当前聊天右侧打开微信式聊天信息栏，再点一次关闭。
- 打开聊天记录栏后再点 `...`，能切到聊天信息栏。
- 单聊信息栏首屏优先显示微信式资料区与高频操作，不再像独立联系人详情页。
- 群聊信息栏首屏优先显示群资料、成员宫格和高频群设置，不再像功能堆叠页。
- 切换会话时，如果右栏处于打开状态，栏位保持打开并刷新成新会话内容。

## 风险与注意事项

- “完全抄微信”的难点不在接口，而在细节节奏：
  - hover 态
  - 分区留白
  - 行高和密度
  - 首屏内容排序
- 当前仓库已经有 2026-04-10 的旧规划，但其中对现状的部分判断已经过时；实施时以本规划和当前代码为准。
- 本次需求不触发模块、实体、路由、表结构变更，因此不需要更新 `AGENTS.md`。
