# 小程序面板 Web 端规划

日期：2026-04-10
范围：`apps/app` Web 手机版 + Web 桌面版

## 目标

- 按微信的信息架构，把“小程序”补成 Web 端可用能力，而不是保留占位页。
- Web 桌面版承接左侧一级 Tab ` /tabs/mini-programs `，形成稳定的小程序工作区。
- Web 手机版从“发现”进入“小程序”二级页，维持微信式“发现 -> 小程序”的浏览节奏。
- 首版不引入独立小程序运行时，不接新后端接口，先把入口、目录、最近使用、我的小程序、搜索和打开态跑通。

## 现状判断

- `apps/app/src/routes/mini-programs-page.tsx` 当前仍是桌面占位工作区。
- 桌面左侧导航已经有“小程序面板”入口，但没有实际功能。
- 移动端“发现”列表还没有“小程序”入口。
- 当前仓库没有小程序目录模型、本地使用状态和桌面/移动双布局组件。

## 对齐微信的范围

- 抄的是微信的小程序入口结构、页面节奏和布局优先级，不是一次性做完微信的小程序生态。
- 首版优先对齐：
  - 最近使用
  - 我的小程序
  - 搜索/筛选
  - 推荐位
  - 分类浏览
  - 打开态承接
- 暂不实现：
  - 真正的小程序运行容器
  - 多窗口管理
  - 服务端同步最近使用
  - 分享到聊天、授权体系、支付能力

## 信息架构

### Web 手机版

- 入口：`发现 -> 小程序`
- 页面结构：
  - 顶部栏：返回、标题、搜索入口
  - 最近使用：微信式轻量网格，优先给常用与最近打开
  - 我的小程序：固定收藏区
  - 推荐专题：更大卡片，承接“继续打开”
  - 分类切换：效率 / 社交 / 内容 / 生活 / 工具
  - 全部小程序：列表卡片，带状态与快捷操作
- 打开行为：首版仍在同页内显示“已打开面板”状态卡，不做真正容器跳转。

### Web 桌面版

- 入口：左侧一级导航 `小程序面板`
- 页面布局：
  - 左栏：分类、最近使用、我的小程序
  - 中栏：搜索、主推荐位、最近使用网格、分类货架
  - 右栏：当前打开态、快捷动作、运营专题、使用提示
- 桌面端更接近微信电脑版“小程序面板”的工作区逻辑，不单独弹新窗口。

## 数据与状态设计

### 共享目录

- 新增 `apps/app/src/features/mini-programs/`
- 目录内提供：
  - 小程序静态目录与分类数据
  - 推荐位、专题、快捷入口 mock
  - 本地持久化工具
  - 移动端共享组件

### 本地持久化

- 使用 `localStorage`
- 维护字段：
  - `activeMiniProgramId`
  - `recentMiniProgramIds`
  - `pinnedMiniProgramIds`
  - `launchCountById`
  - `lastOpenedAtById`
- 触发时机：
  - 点击“打开 / 继续使用”时写入最近使用
  - 点击“加入我的小程序”时更新 pinned
  - 点击“关闭当前面板”时仅清理 active

## 路由与结构变更

- 保留 `mini-programs-page.tsx` 作为统一容器，根据布局切换移动/桌面视图。
- 新增移动端发现子路由：`/discover/mini-programs`
- 更新 `discover-page.tsx` 的移动端入口列表。
- 因新增 `features/mini-programs/` 目录与发现子路由，需要同步更新 `AGENTS.md`。

## 实现分步

1. 新增规划文档并提交。
2. 新增共享小程序目录数据、本地状态存储和状态 hook。
3. 把 `mini-programs-page.tsx` 改造成统一页面容器。
4. 新增桌面工作区组件，完成微信式三栏布局。
5. 新增移动端小程序页，补齐最近使用、我的小程序、推荐位和分类列表。
6. 在路由与发现入口接入 `/discover/mini-programs`。
7. 更新 `AGENTS.md` 里的发现页、小程序页和产品口径描述。
8. 执行 `pnpm --filter @yinjie/app build` 验证。

## 首版边界

- 不生成测试文件。
- 不新增后端接口。
- 不做真实小程序 WebView/iframe 容器。
- 不改动现有聊天、公众号、视频号和游戏中心的数据流。

## 后续演进

- 接入服务端小程序目录和运营位编排。
- 增加最近使用云同步与多端同步。
- 支持在聊天页、发现页、收藏页复用小程序卡片。
- 评估桌面端独立工作区、多开、外部窗口和真正容器运行策略。
