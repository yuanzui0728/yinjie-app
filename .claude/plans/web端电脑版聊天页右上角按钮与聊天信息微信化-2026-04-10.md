# Web 端电脑版聊天页右上角按钮与聊天信息微信化规划

## 任务目标
- 将 `apps/app` 的 Web 电脑版聊天工作区右上角入口改成当前微信电脑端的三按钮结构：
  - `聊天记录`
  - `通话`
  - `更多`
- 点击三个点后，不再使用当前项目的泛化信息抽屉，而是改成更接近微信电脑端的右侧聊天信息面板。
- 范围同时覆盖：
  - 单聊：`/chat/$conversationId`
  - 群聊：`/group/$groupId`

## 参考基线

### 公开信息确认到的部分
- 2026-03-06 的公开体验文章显示，微信 Mac / Windows `4.1.8` 版本已将 `语音 / 视频通话` 入口移动到聊天界面右上角。
- 同一轮改版中，聊天输入区的快捷按钮被移到输入框左下方，聊天背景趋向纯白，整体视觉更扁平。

### 需要明确标注的推断
- 公开文章能确认“通话入口右移”和整体界面收平，但没有逐项清晰列出右上角每个按钮和右侧信息面板的全部字段。
- 因此本规划对 `聊天记录按钮` 和 `三个点后的信息面板结构`，采用两条依据合并：
  - 用户指定目标：要做成“微信现在的”电脑端聊天页
  - 微信电脑端长期稳定交互：右侧承载聊天记录 / 聊天信息相关侧栏，而不是跳到独立全页

## 当前现状

### 1. 右上角按钮不对
- `apps/app/src/features/chat/conversation-thread-panel.tsx`
  - 桌面单聊头部目前是 `搜索 / 语音通话 / 视频通话 / 通讯录 / 更多`
  - 这套组合明显不是当前微信电脑端
- `apps/app/src/features/chat/group-chat-thread-panel.tsx`
  - 桌面群聊头部没有统一右上角动作区，只展示标题和成员条

### 2. 三个点后的承载方式不对
- `apps/app/src/features/desktop/chat/desktop-chat-workspace.tsx`
  - 当前用 `inspectorOpen` 控制一个通用右侧抽屉
  - 抽屉内容只有头像、类型、成员数、最后活跃、跳转入口，信息密度太低
  - 行为更像项目自定义“检查器”，不是微信电脑端聊天信息面板

### 3. 已有能力能复用，但入口形态不对
- 聊天记录：
  - 单聊 `apps/app/src/routes/chat-message-search-page.tsx`
  - 群聊 `apps/app/src/routes/group-message-search-page.tsx`
  - 目前是移动端二级页，不适合直接拿来当电脑端右侧面板
- 聊天信息：
  - 单聊 `apps/app/src/routes/chat-details-page.tsx`
  - 群聊 `apps/app/src/routes/group-chat-details-page.tsx`
  - 目前是移动端全页壳，结构能参考，但视觉和交互不适合电脑端
- 全局聊天记录管理：
  - `apps/app/src/routes/desktop-chat-history-page.tsx`
  - 这是独立管理页，不是当前聊天上下文中的微信式右侧记录面板

### 4. 整体视觉也还没到微信电脑端
- 当前桌面聊天页仍大量使用米色渐变、圆角按钮、悬浮卡片感。
- 如果只改三个按钮，不改头部和右侧面板的表面样式，结果不会像微信。

## 目标口径

### 1. 右上角只保留三项
- 单聊和群聊统一为三个入口：
  - `聊天记录`
  - `通话`
  - `更多`
- 删除当前不符合微信的入口：
  - `搜索`
  - 独立 `视频通话`
  - `通讯录`

### 2. 通话按钮改成单一入口
- 用一个按钮承载通话能力。
- 点击后弹出一个小型菜单：
  - `语音通话`
  - `视频通话`
- 这比当前“语音 / 视频分两个按钮”更接近微信近期桌面端结构。

### 3. 右侧面板改成两种模式
- 不再只存在一个“更多信息抽屉”。
- 改为统一的桌面聊天侧栏容器，支持两种模式：
  - `history`：聊天记录面板
  - `details`：聊天信息面板
- 右上角两个按钮分别打开对应模式。

### 4. 三个点承载的是右侧聊天信息面板，不是全页跳转
- 桌面端继续留在当前聊天上下文中。
- 点击三个点后，右侧展开微信式聊天信息面板。
- 这比跳到 `/chat/$conversationId/details` 或 `/group/$groupId/details` 更符合电脑端微信形态。

## 目标信息架构

### A. 单聊聊天记录面板
- 顶部：
  - 标题 `聊天记录`
  - 当前会话名
  - 关闭按钮
- 主体：
  - 搜索输入框
  - 匹配结果列表
  - 每项展示：发送者、时间、命中片段
  - 点击结果后回到当前会话消息区并滚动到目标消息
- 首版数据来源：
  - 直接复用 `getConversationMessages`
  - 采用当前 `chat-message-search-page.tsx` 的本地过滤逻辑

### B. 群聊聊天记录面板
- 与单聊一致，但数据源换成 `getGroupMessages`
- 副标题显示群名
- 列表项继续展示发送者与命中片段

### C. 单聊聊天信息面板
- 顶部资料区：
  - 头像
  - 名称
  - 关系摘要
- 常用入口区：
  - `查找聊天记录`
  - `聊天文件`
- 设置区：
  - `消息免打扰`
  - `置顶聊天`
  - `设置当前聊天背景`
- 风险区：
  - `清空聊天记录`
  - `投诉`
  - `加入黑名单`

### D. 群聊聊天信息面板
- 顶部资料区：
  - 群名
  - 成员数
- 成员区：
  - 微信式成员头像栅格
  - `+` / `-` 入口保留
- 常用入口区：
  - `查找聊天记录`
  - `聊天文件`
- 设置区：
  - `消息免打扰`
  - `置顶聊天`
  - `我在本群的昵称`
  - `显示群成员昵称`
  - `设置当前聊天背景`
- 风险区：
  - `清空聊天记录`
  - `退出群聊`

## 功能映射

### 已有能力，可直接接通
- 单聊：
  - `setConversationPinned`
  - `clearConversationHistory`
  - `createModerationReport`
  - `blockCharacter`
- 群聊：
  - `setGroupPinned`
  - `clearGroupMessages`
  - `leaveGroup`
  - `updateGroupOwnerProfile`
- 搜索：
  - 单聊 `getConversationMessages`
  - 群聊 `getGroupMessages`
- 聊天背景：
  - 单聊已有 `/chat/$conversationId/background`
  - 群聊当前走默认背景语义，首版可以只保留入口或沿用现有 group 背景处理

### 只能先做微信式入口或本地偏好的部分
- `消息免打扰`
  - 当前没有完整桌面端会话免打扰模型
  - 首版建议按 `conversationId/groupId` 走本地偏好
- `通话`
  - 当前仓库没有 RTC / WebRTC / 呼叫链路
  - 首版只能做微信式入口和菜单，不做真实通话
- `聊天文件`
  - 当前已有全局页 `/desktop/chat-files`
  - 但没有按当前会话筛选的电脑端侧栏
  - 首版建议保留入口位或跳全局页，二期再做会话上下文过滤

## 推荐实现方案

### Phase 1：重做桌面头部动作区
目标：先把最显眼的入口改对。

改造点：
- `apps/app/src/features/chat/conversation-thread-panel.tsx`
  - 桌面头部改成三按钮结构
- `apps/app/src/features/chat/group-chat-thread-panel.tsx`
  - 补齐和单聊一致的三按钮头部
- 建议抽出桌面头部动作组件：
  - `apps/app/src/features/desktop/chat/desktop-chat-header-actions.tsx`

实现要求：
- 按钮视觉收平，不再使用当前“悬浮圆形工具按钮”风格
- 图标和间距接近微信电脑端
- 通话按钮是一个入口，不再拆成两个

### Phase 2：把右侧抽屉升级成微信式双模式侧栏
目标：让 `聊天记录` 和 `更多` 都有正确承载。

核心改造：
- `apps/app/src/features/desktop/chat/desktop-chat-workspace.tsx`
  - 把 `inspectorOpen: boolean` 升级为：
    - `rightPanelMode: "history" | "details" | null`
  - 当前右侧 aside 改成统一侧栏容器

建议新增：
- `apps/app/src/features/desktop/chat/desktop-chat-side-panel.tsx`
- `apps/app/src/features/desktop/chat/desktop-chat-history-panel.tsx`
- `apps/app/src/features/desktop/chat/desktop-chat-details-panel.tsx`

交互规则：
- 点 `聊天记录`：
  - 若当前已打开 `history`，再次点击则关闭
  - 否则切到 `history`
- 点 `更多`：
  - 若当前已打开 `details`，再次点击则关闭
  - 否则切到 `details`
- 切换会话时：
  - 右侧面板保持打开，但内容随当前会话切换
  - 这比每次强行关闭更接近电脑端工作流

### Phase 3：提炼单聊 / 群聊聊天信息面板
目标：不再复用移动端详情页壳，但复用已有业务能力。

建议做法：
- 把移动端详情页中的数据读取和 mutation 逐步抽离成可复用 hook
- 电脑端面板只复用能力，不复用页面外壳

建议新增 hook：
- `apps/app/src/features/chat-details/use-direct-chat-details.ts`
- `apps/app/src/features/chat-details/use-group-chat-details.ts`

面板内组件建议：
- `desktop-chat-details-section.tsx`
- `desktop-chat-details-row.tsx`
- `desktop-chat-member-grid.tsx`

### Phase 4：补一个微信式通话菜单
目标：入口先对，真实通话后补。

建议新增：
- `apps/app/src/features/desktop/chat/desktop-chat-call-menu.tsx`

首版行为：
- 点击通话按钮展开小菜单：
  - `语音通话`
  - `视频通话`
- 由于当前无 RTC，点击后提示：
  - `当前先保留微信式入口，真实通话能力后续补齐`

后续若要做真功能，需要新增：
- 通话状态模型
- 呼叫邀请接口
- WebRTC 媒体与设备权限处理
- 单聊 / 群聊通话差异处理

## 视觉要求

### 1. 桌面聊天头部
- 用更平的白底或极浅灰底
- 仅保留细分隔线
- 按钮 hover 轻灰，不用品牌色大面积发光
- 标题区和按钮区留白贴近微信电脑端

### 2. 右侧面板
- 宽度控制在 `300px - 340px`
- 白底或浅灰底
- 分组之间主要靠间距和细线，不靠大圆角卡片
- 成员区、设置区、危险区保持微信式垂直阅读节奏

### 3. 与当前界面的联动收口
- 如果头部和侧栏都微信化，主聊天区仍保留暖色渐变，会显得割裂。
- 建议同步把桌面聊天头部下方的大背景和容器表面也收平：
  - 聊天主背景趋向白 / 极浅灰
  - 输入区与消息区边界更明确
- 这一步不是本需求的唯一重点，但不做的话最终效果会明显“不像微信”。

## 风险与取舍

### 1. 不能只改图标，不改承载层
- 如果只把按钮换成三个图标，但保留现在的简陋 inspector，用户一眼就会看出不是微信。

### 2. 通话只能先做入口
- 当前仓库没有真实通话链路。
- 如果用户要求“完全抄微信”包含可发起通话，那将是单独一条功能线，不属于纯前端排版改造。

### 3. 聊天记录要做成当前会话上下文
- 现有 `/desktop/chat-history` 是全局页。
- 本次如果继续跳这个全局页，会破坏微信电脑端的聊天内连续性。

### 4. 单聊 / 群聊不能继续分裂两套桌面头部
- 这次应该把桌面聊天头部动作抽象成统一实现。
- 否则后续每补一个按钮都要改两份。

## 实施顺序
1. 先统一单聊 / 群聊桌面头部动作区，只保留三按钮。
2. 再把 `desktop-chat-workspace` 的右侧抽屉改成双模式侧栏。
3. 先完成 `聊天记录` 面板，因为已有搜索逻辑可直接复用。
4. 再完成 `更多` 面板，按单聊 / 群聊分别映射已有能力。
5. 最后补通话菜单和样式收平。

## 预期结果
- 进入桌面聊天页后，右上角只会看到微信电脑端语义明确的三按钮。
- `聊天记录` 不再跳全局管理页，而是在当前聊天内打开右侧记录面板。
- `更多` 不再打开项目自定义检查器，而是打开微信式聊天信息面板。
- 单聊和群聊的桌面头部交互被统一，后续维护成本显著下降。
